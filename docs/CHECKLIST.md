# Checklist

## Before you start
- [ ] Create a new branch (never work directly on `main`).
- [ ] Confirm the scope is small and focused.
- [ ] Ensure no secrets or credentials will be added.

## Before you open a PR
- [ ] Run `scripts/verify.sh` and review the output.
- [ ] Perform manual smoke checks for the changes made.

## Before merging
- [ ] Verify required disclosure copy was not removed or altered.
- [ ] If rewrites changed, note any permalink flush requirements.
